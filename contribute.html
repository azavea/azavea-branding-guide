--- layout: page title: Contribute to the guide description: This page explains the process by which the design system gets updated. status: in_progress --- We encourage Azavea staff to contribute to the Azavea Branding Guide to document issues or recommend additions. As a first step, reach out to us via the [#marketing channel](https://slack.com/app_redirect?channel=marketing) in the Azavea Slack if you would like to ask questions, voice concerns, or propose improvements to this document. You can also contribute directly to the project by submitting a Pull Request on GitHub. GitHub repository: [https://github.com/azavea/open-data-standards](https://github.com/azavea/open-data-standards) Read the contribution guidelines below before contributing directly to the project. ## How to Contribute ### Git Branching Model This site follows the standard practice of using the master branch as main integration branch. ### Git Commit Messages We follow the 'imperative present tense' style for commit messages (e.g. "Add color palate download"). ### Issue Tracking If you find an error in the content or missing information, please create an Issue. ### Pull Requests If you'd like to submit a code contribution please submit a Pull Request against the Master branch. Like any other project, the Marketing Team might ask you to go through some discussion and refinement before merging. ## Ways to Contribute - document issues - propose a change to existing content - propose the addition of new content
